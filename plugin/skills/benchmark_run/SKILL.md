# Benchmark Runner

Executor and watchdog for benchmark runs. Launches self-contained scripts that survive session death, monitors via `/loop`, and triggers `/benchmark_agent analyze` on completion.

**This skill NEVER makes analytical decisions.** It only answers: "Is it running? Is it done? Is it in trouble?"

## Actions

### `<run_id>` — Launch a benchmark
Requires: approved plan at `benchmark_data/plans/plan_<run_id>.md` AND generated scripts at `benchmark_data/scripts/`

### `resume` — Resume monitoring an in-flight run
Detects active runs (monitor process alive, no completion signal) and restarts the `/loop`.

### `status` — Quick status check
Shows: active runs, recent completions, any alerts.

## Execution Steps

### Step 1: Verify Preconditions

```bash
RUN_ID="<from argument>"
BENCH_HOME="${BENCHMARK_HOME:-./benchmark_data}"
SCRIPTS_DIR="$BENCH_HOME/scripts"
MONITORS_DIR="$BENCH_HOME/monitors"

# Verify plan exists and is approved
[ -f "$BENCH_HOME/plans/plan_${RUN_ID}.md" ] || { echo "ERROR: No approved plan for $RUN_ID"; exit 1; }

# Verify scripts were generated by /benchmark_agent
[ -f "$SCRIPTS_DIR/guardrail_${RUN_ID}.sh" ] || { echo "ERROR: No guardrail script"; exit 1; }
[ -f "$SCRIPTS_DIR/execute_${RUN_ID}.sh" ] || { echo "ERROR: No execute script"; exit 1; }
[ -f "$SCRIPTS_DIR/monitor_${RUN_ID}.sh" ] || { echo "ERROR: No monitor script"; exit 1; }
[ -f "$SCRIPTS_DIR/check_${RUN_ID}.sh" ] || { echo "ERROR: No check script"; exit 1; }
```

If any script is missing, tell the user to run `/benchmark_agent plan <type>` first.

### Step 2: Pre-Launch Verification

Before launching, do a final sanity check:
```bash
# Make scripts executable
chmod +x "$SCRIPTS_DIR/guardrail_${RUN_ID}.sh"
chmod +x "$SCRIPTS_DIR/execute_${RUN_ID}.sh"
chmod +x "$SCRIPTS_DIR/monitor_${RUN_ID}.sh"
chmod +x "$SCRIPTS_DIR/check_${RUN_ID}.sh"

# Verify target systems reachable (read from plan)
# This is benchmark-specific — check the plan for target nodes

# Verify no other benchmark is currently running
ls "$MONITORS_DIR"/*.running 2>/dev/null && { echo "ERROR: Another benchmark is active"; exit 1; }

# Create running signal
touch "$MONITORS_DIR/${RUN_ID}.running"
```

### Step 3: Launch Guardrail (First — Non-Negotiable)

The hard ceiling guardrail MUST be the first process started. It protects the node from OOM regardless of what the benchmark does.

```bash
nohup bash "$SCRIPTS_DIR/guardrail_${RUN_ID}.sh" > "$MONITORS_DIR/${RUN_ID}_guardrail_stdout.log" 2>&1 &
GUARDRAIL_PID=$!
echo "$GUARDRAIL_PID" > "$MONITORS_DIR/${RUN_ID}_guardrail.pid"

# Verify it started
sleep 2
if ! kill -0 "$GUARDRAIL_PID" 2>/dev/null; then
  echo "ERROR: Guardrail script died immediately — CANNOT proceed without node protection"
  cat "$MONITORS_DIR/${RUN_ID}_guardrail_stdout.log"
  rm -f "$MONITORS_DIR/${RUN_ID}.running"
  exit 1
fi
echo "Guardrail active (PID $GUARDRAIL_PID) — node protected at 85% RAM ceiling"
```

**If the guardrail fails to start, the benchmark MUST NOT proceed.** No exceptions.

### Step 4: Launch Monitor

Start the resource monitor BEFORE the execution script — it must be capturing baseline.

```bash
nohup bash "$SCRIPTS_DIR/monitor_${RUN_ID}.sh" > "$MONITORS_DIR/${RUN_ID}_monitor_stdout.log" 2>&1 &
MONITOR_PID=$!
echo "$MONITOR_PID" > "$MONITORS_DIR/${RUN_ID}_monitor.pid"

# Verify it started
sleep 2
if ! kill -0 "$MONITOR_PID" 2>/dev/null; then
  echo "ERROR: Monitor script died immediately"
  cat "$MONITORS_DIR/${RUN_ID}_monitor_stdout.log"
  rm -f "$MONITORS_DIR/${RUN_ID}.running"
  exit 1
fi
echo "Monitor started (PID $MONITOR_PID)"
```

### Step 5: Launch Execution

```bash
nohup bash "$SCRIPTS_DIR/execute_${RUN_ID}.sh" > "$MONITORS_DIR/${RUN_ID}_execute_stdout.log" 2>&1 &
EXEC_PID=$!
echo "$EXEC_PID" > "$MONITORS_DIR/${RUN_ID}_execute.pid"

# Verify it started
sleep 2
if ! kill -0 "$EXEC_PID" 2>/dev/null; then
  echo "ERROR: Execute script died immediately"
  cat "$MONITORS_DIR/${RUN_ID}_execute_stdout.log"
  rm -f "$MONITORS_DIR/${RUN_ID}.running"
  exit 1
fi
echo "Execution started (PID $EXEC_PID)"
```

### Step 6: Start Loop

Start `/loop` with the check script. The loop prompt should:
1. Run the check script
2. Interpret the output
3. Take action based on status

```
/loop 5m Benchmark monitor for run <RUN_ID>. Run: bash $BENCHMARK_HOME/scripts/check_<RUN_ID>.sh. Then interpret the output:
- If STATUS: COMPLETE → stop this loop (use CronDelete), invoke /benchmark_agent analyze <RUN_ID>, remove the .running file
- If STATUS: FAILED → stop this loop, report failure details to user, remove the .running file  
- If STATUS: MONITOR_DEAD → try to restart: nohup bash $BENCHMARK_HOME/scripts/monitor_<RUN_ID>.sh > /dev/null 2>&1 &. If restart fails, alert user.
- If any CRITICAL lines → alert user with details, suggest corrective action
- If STATUS: RUNNING with all OK → report one line: phase, peak RSS, time elapsed
```

### Step 7: Confirm Launch

Report to user:
```
Benchmark <RUN_ID> launched successfully.
  Guardrail PID: <pid> (nohup, kills at 85% RAM)
  Execution PID: <pid> (nohup, survives session)
  Monitor PID: <pid> (nohup, survives session)
  Loop: checking every 5 minutes
  
  Scripts: benchmark_data/scripts/{guardrail,execute,monitor,check}_<RUN_ID>.sh
  Logs: benchmark_data/monitors/<RUN_ID>_monitor.log
  Events: benchmark_data/monitors/<RUN_ID>_events.log

Node protection: guardrail will SIGTERM the benchmark if RSS > 85% physical RAM sustained >10s.
You can safely close this session. On return, use /benchmark_run resume or /benchmark_run status.
```

## Resume Protocol

When invoked with `resume`:

```bash
BENCH_HOME="${BENCHMARK_HOME:-./benchmark_data}"

# Find active runs
for running_file in "$BENCH_HOME/monitors"/*.running; do
  [ -f "$running_file" ] || continue
  RUN_ID=$(basename "$running_file" .running)
  
  # Check if it's actually still running
  if [ -f "$BENCH_HOME/monitors/${RUN_ID}.complete" ]; then
    echo "$RUN_ID: COMPLETED (analysis pending)"
    rm -f "$running_file"
  elif [ -f "$BENCH_HOME/monitors/${RUN_ID}.failed" ]; then
    echo "$RUN_ID: FAILED (investigation needed)"
    rm -f "$running_file"
  else
    # Still running — check guardrail FIRST (critical), then monitor
    GUARDRAIL_PID=$(cat "$BENCH_HOME/monitors/${RUN_ID}_guardrail.pid" 2>/dev/null)
    if [ -n "$GUARDRAIL_PID" ] && kill -0 "$GUARDRAIL_PID" 2>/dev/null; then
      echo "$RUN_ID: guardrail alive (PID $GUARDRAIL_PID)"
    else
      echo "$RUN_ID: GUARDRAIL DEAD — restarting IMMEDIATELY"
      nohup bash "$BENCH_HOME/scripts/guardrail_${RUN_ID}.sh" > "$BENCH_HOME/monitors/${RUN_ID}_guardrail_stdout.log" 2>&1 &
      echo $! > "$BENCH_HOME/monitors/${RUN_ID}_guardrail.pid"
    fi
    
    MONITOR_PID=$(cat "$BENCH_HOME/monitors/${RUN_ID}_monitor.pid" 2>/dev/null)
    if [ -n "$MONITOR_PID" ] && kill -0 "$MONITOR_PID" 2>/dev/null; then
      echo "$RUN_ID: RUNNING (monitor alive, PID $MONITOR_PID)"
    else
      echo "$RUN_ID: RUNNING but monitor DEAD — restarting"
      nohup bash "$BENCH_HOME/scripts/monitor_${RUN_ID}.sh" > "$BENCH_HOME/monitors/${RUN_ID}_monitor_stdout.log" 2>&1 &
      echo $! > "$BENCH_HOME/monitors/${RUN_ID}_monitor.pid"
    fi
    # Restart loop for this run
    echo "Restarting /loop for $RUN_ID"
    # → invoke /loop 5m with the check script (same as Step 6)
  fi
done
```

If a completed run is found, immediately trigger: `/benchmark_agent analyze <RUN_ID>`

## Status Check Protocol

When invoked with `status`:

```bash
BENCH_HOME="${BENCHMARK_HOME:-./benchmark_data}"

echo "=== Benchmark Runner Status ==="

# Active runs
echo "--- Active Runs ---"
for f in "$BENCH_HOME/monitors"/*.running; do
  [ -f "$f" ] || { echo "None"; break; }
  RUN_ID=$(basename "$f" .running)
  # Run the check script for quick status
  bash "$BENCH_HOME/scripts/check_${RUN_ID}.sh" 2>/dev/null | head -10
done

# Recent completions (last 5)
echo "--- Recent Completions ---"
ls -t "$BENCH_HOME/monitors"/*.complete 2>/dev/null | head -5 | while read f; do
  RUN_ID=$(basename "$f" .complete)
  echo "$RUN_ID (completed $(stat -c %y "$f" | cut -d. -f1))"
done

# Recent failures
echo "--- Recent Failures ---"
ls -t "$BENCH_HOME/monitors"/*.failed 2>/dev/null | head -5 | while read f; do
  RUN_ID=$(basename "$f" .failed)
  echo "$RUN_ID (failed $(stat -c %y "$f" | cut -d. -f1))"
done
```

## Loop Decision Logic

The `/loop` check-in interprets the check script output:

| Check Script Output | Loop Action |
|---|---|
| `STATUS: RUNNING`, all `OK` | One-line report: "Run X: phase Y, peak Z GB, elapsed T." |
| `STATUS: RUNNING`, has `WARNING` | Report: "WARNING on node X: RSS at Y%. Still within limits. Monitoring." |
| `STATUS: RUNNING`, has `CRITICAL` | Alert user. Include node, metric, value. Suggest: pause merge? kill background task? |
| `STATUS: COMPLETE` | Stop loop. Remove `.running`. Invoke `/benchmark_agent analyze <id>`. |
| `STATUS: FAILED` | Stop loop. Remove `.running`. Report: failure phase, last metrics, crash artifacts. |
| `STATUS: FAILED` + `GUARDRAIL_KILL` in events | Stop loop. Remove `.running`. Report: "Guardrail killed benchmark at X GB — node survived. Run classified UNRELIABLE." |
| `STATUS: MONITOR_DEAD` | Attempt restart. If fails, alert user: "Monitor died, cannot guarantee data collection." |
| `STATUS: GUARDRAIL_DEAD` | CRITICAL alert: "Node protection lost — guardrail process died." Attempt restart from script. |
| Check script itself fails | Alert user: "Check script error — investigate manually." |

## Script Templates

The `/benchmark_agent` generates these scripts during planning. Templates below show the structure — the agent fills in specifics from the plan.

### guardrail_<id>.sh template

```bash
#!/bin/bash
# Hard ceiling guardrail — kills benchmark if RSS exceeds safe limits
# Generated by /benchmark_agent, deployed FIRST by /benchmark_run
set -uo pipefail
RUN_ID="<id>"
BENCH_HOME="${BENCHMARK_HOME:-./benchmark_data}"
LOG_DIR="$BENCH_HOME/monitors"
EXEC_PID_FILE="$LOG_DIR/${RUN_ID}_execute.pid"
EVENTS_LOG="$LOG_DIR/${RUN_ID}_events.log"
SIGNAL_COMPLETE="$LOG_DIR/${RUN_ID}.complete"
SIGNAL_FAILED="$LOG_DIR/${RUN_ID}.failed"

# Thresholds (from resource budget)
TOTAL_RAM_KB=<total_ram_kb>
WARN_KB=$((TOTAL_RAM_KB * 75 / 100))
HARD_CEILING_KB=$((TOTAL_RAM_KB * 85 / 100))
EMERGENCY_KB=$((TOTAL_RAM_KB * 92 / 100))
POLL_INTERVAL=5

WARN_COUNT=0
HARD_COUNT=0
WARN_SUSTAINED=6   # 30s
HARD_SUSTAINED=2   # 10s

log_event() { echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) GUARDRAIL: $1" >> "$EVENTS_LOG"; }

get_target_rss() {
  # Adapt per benchmark — measure the process that consumes memory
  <rss_measurement_command>
}

log_event "GUARDRAIL_START: ceiling=${HARD_CEILING_KB}KB (85% of ${TOTAL_RAM_KB}KB)"

while true; do
  [ -f "$SIGNAL_COMPLETE" ] && { log_event "GUARDRAIL_EXIT: benchmark completed normally"; exit 0; }
  [ -f "$SIGNAL_FAILED" ] && { log_event "GUARDRAIL_EXIT: benchmark already failed"; exit 0; }

  # Check if execution process still alive
  if [ -f "$EXEC_PID_FILE" ]; then
    EXEC_PID=$(cat "$EXEC_PID_FILE")
    if ! kill -0 "$EXEC_PID" 2>/dev/null; then
      log_event "GUARDRAIL_EXIT: exec process gone (PID $EXEC_PID)"
      exit 0
    fi
  fi

  RSS_KB=$(get_target_rss)

  if [ "$RSS_KB" -gt "$EMERGENCY_KB" ]; then
    log_event "EMERGENCY_KILL: RSS=${RSS_KB}KB > ${EMERGENCY_KB}KB — SIGKILL immediate"
    kill -9 "$EXEC_PID" 2>/dev/null
    touch "$SIGNAL_FAILED"
    exit 1

  elif [ "$RSS_KB" -gt "$HARD_CEILING_KB" ]; then
    HARD_COUNT=$((HARD_COUNT + 1))
    if [ "$HARD_COUNT" -ge "$HARD_SUSTAINED" ]; then
      log_event "HARD_KILL: RSS=${RSS_KB}KB > ${HARD_CEILING_KB}KB sustained ${HARD_COUNT}×${POLL_INTERVAL}s — SIGTERM"
      kill -15 "$EXEC_PID" 2>/dev/null
      sleep 10
      kill -0 "$EXEC_PID" 2>/dev/null && kill -9 "$EXEC_PID" 2>/dev/null
      touch "$SIGNAL_FAILED"
      exit 1
    fi

  elif [ "$RSS_KB" -gt "$WARN_KB" ]; then
    HARD_COUNT=0
    WARN_COUNT=$((WARN_COUNT + 1))
    if [ "$WARN_COUNT" -ge "$WARN_SUSTAINED" ]; then
      log_event "WARN_SUSTAINED: RSS=${RSS_KB}KB > ${WARN_KB}KB for $((WARN_COUNT * POLL_INTERVAL))s"
      WARN_COUNT=0
    fi

  else
    WARN_COUNT=0
    HARD_COUNT=0
  fi

  sleep "$POLL_INTERVAL"
done
```

### monitor_<id>.sh template

```bash
#!/bin/bash
set -uo pipefail
RUN_ID="<id>"
BENCH_HOME="${BENCHMARK_HOME:-./benchmark_data}"
LOG_DIR="$BENCH_HOME/monitors"
LOG_FILE="$LOG_DIR/${RUN_ID}_monitor.log"
SIGNAL_COMPLETE="$LOG_DIR/${RUN_ID}.complete"
SIGNAL_FAILED="$LOG_DIR/${RUN_ID}.failed"

# Configuration (filled by agent from plan)
NODES=(<node_list>)
SSH_KEY="<ssh_key_path>"
TOTAL_RAM_KB=<total_ram>
WARN_KB=$((TOTAL_RAM_KB * <warn_pct> / 100))
CRIT_KB=$((TOTAL_RAM_KB * <crit_pct> / 100))
INTERVAL=<interval_seconds>

get_rss() {
  local node=$1
  ssh -i "$SSH_KEY" -o StrictHostKeyChecking=no -o ConnectTimeout=5 "$node" \
    "<rss_command>" 2>/dev/null || echo "0"
}

get_phase() {
  local events="$LOG_DIR/${RUN_ID}_events.log"
  grep "PHASE_" "$events" 2>/dev/null | tail -1 | sed 's/.*PHASE_[A-Z]*: //' || echo "unknown"
}

echo "timestamp,node,rss_kb,phase,status" > "$LOG_FILE"

while true; do
  [ -f "$SIGNAL_COMPLETE" ] && exit 0
  [ -f "$SIGNAL_FAILED" ] && exit 1

  PHASE=$(get_phase)
  for node in "${NODES[@]}"; do
    RSS_KB=$(get_rss "$node")
    if [ "$RSS_KB" -gt "$CRIT_KB" ]; then
      STATUS="CRITICAL"
    elif [ "$RSS_KB" -gt "$WARN_KB" ]; then
      STATUS="WARNING"
    else
      STATUS="OK"
    fi
    echo "$(date +%s),$node,$RSS_KB,$PHASE,$STATUS" >> "$LOG_FILE"
  done
  sleep "$INTERVAL"
done
```

### check_<id>.sh template

```bash
#!/bin/bash
RUN_ID="<id>"
BENCH_HOME="${BENCHMARK_HOME:-./benchmark_data}"
LOG_DIR="$BENCH_HOME/monitors"
LOG_FILE="$LOG_DIR/${RUN_ID}_monitor.log"
SIGNAL_COMPLETE="$LOG_DIR/${RUN_ID}.complete"
SIGNAL_FAILED="$LOG_DIR/${RUN_ID}.failed"
EVENTS_LOG="$LOG_DIR/${RUN_ID}_events.log"

echo "=== Benchmark Check: $RUN_ID ($(date -u +%H:%M:%S)) ==="

# --- Status ---
if [ -f "$SIGNAL_COMPLETE" ]; then
  echo "STATUS: COMPLETE"
  echo "COMPLETED_AT: $(stat -c %Y "$SIGNAL_COMPLETE")"
elif [ -f "$SIGNAL_FAILED" ]; then
  echo "STATUS: FAILED"
  echo "FAILED_AT: $(stat -c %Y "$SIGNAL_FAILED")"
  echo "LAST_EVENT: $(tail -1 "$EVENTS_LOG" 2>/dev/null)"
elif [ -f "$LOG_DIR/${RUN_ID}_monitor.pid" ] && kill -0 "$(cat "$LOG_DIR/${RUN_ID}_monitor.pid")" 2>/dev/null; then
  echo "STATUS: RUNNING"
else
  echo "STATUS: MONITOR_DEAD"
fi

# --- Guardrail Status ---
echo "--- Guardrail ---"
if [ -f "$LOG_DIR/${RUN_ID}_guardrail.pid" ]; then
  GUARDRAIL_PID=$(cat "$LOG_DIR/${RUN_ID}_guardrail.pid")
  if kill -0 "$GUARDRAIL_PID" 2>/dev/null; then
    echo "GUARDRAIL: ACTIVE (PID $GUARDRAIL_PID)"
  else
    echo "GUARDRAIL: DEAD — NODE UNPROTECTED"
    echo "STATUS: GUARDRAIL_DEAD"
  fi
else
  echo "GUARDRAIL: NO_PID_FILE"
fi

# --- Elapsed Time ---
if [ -f "$EVENTS_LOG" ]; then
  START_TIME=$(head -1 "$EVENTS_LOG" | cut -d' ' -f1)
  echo "STARTED: $START_TIME"
  echo "ELAPSED: $(( $(date +%s) - $(date -d "$START_TIME" +%s 2>/dev/null || echo $(date +%s)) ))s"
fi

# --- Current Phase ---
echo "--- Phase ---"
grep "PHASE_" "$EVENTS_LOG" 2>/dev/null | tail -3

# --- Latest Metrics (last entry per node) ---
echo "--- Latest Metrics ---"
if [ -f "$LOG_FILE" ]; then
  # Get unique nodes from header-less data
  tail -n +2 "$LOG_FILE" | awk -F',' '{print $2}' | sort -u | while read node; do
    grep ",$node," "$LOG_FILE" | tail -1
  done
fi

# --- Peak RSS ---
echo "--- Peak RSS (KB) ---"
if [ -f "$LOG_FILE" ]; then
  tail -n +2 "$LOG_FILE" | awk -F',' '{
    if ($3 > peak[$2]) peak[$2] = $3
  } END {
    for (n in peak) print n ": " peak[n]
  }'
fi

# --- Recent Alerts (last 10 min) ---
echo "--- Alerts (last 10m) ---"
if [ -f "$LOG_FILE" ]; then
  CUTOFF=$(($(date +%s) - 600))
  tail -n +2 "$LOG_FILE" | awk -F',' -v cutoff="$CUTOFF" \
    '$1 > cutoff && ($5 == "WARNING" || $5 == "CRITICAL")' | tail -5
  ALERT_COUNT=$(tail -n +2 "$LOG_FILE" | awk -F',' -v cutoff="$CUTOFF" \
    '$1 > cutoff && ($5 == "WARNING" || $5 == "CRITICAL")' | wc -l)
  echo "ALERT_COUNT_10M: $ALERT_COUNT"
else
  echo "ALERT_COUNT_10M: 0"
fi
```

## Integration with /benchmark_agent

The agent and runner interact through the filesystem:

```
/benchmark_agent plan ...
  → writes: benchmark_data/plans/plan_<id>.md
  → writes: benchmark_data/scripts/guardrail_<id>.sh
  → writes: benchmark_data/scripts/execute_<id>.sh
  → writes: benchmark_data/scripts/monitor_<id>.sh
  → writes: benchmark_data/scripts/check_<id>.sh

/benchmark_run <id>
  → reads: plans + scripts (verifies they exist)
  → creates: monitors/<id>.running, *.pid files
  → starts: nohup processes + /loop

[loop detects completion]
  → invokes: /benchmark_agent analyze <id>
  → removes: monitors/<id>.running

/benchmark_agent analyze <id>
  → reads: monitors/<id>_monitor.log, <id>_events.log
  → writes: runs/run_<id>.md (full report)
  → updates: results_index.md
  → appends: lessons.md (if new)
```

## Session Death and Recovery

The runner is designed for sessions to die mid-run:

| Component | Survives session? | Recovery on next session |
|-----------|-------------------|------------------------|
| `guardrail_<id>.sh` | Yes (nohup) | Check PID file, restart if dead — CRITICAL |
| `execute_<id>.sh` | Yes (nohup) | Check PID file, or completion signal |
| `monitor_<id>.sh` | Yes (nohup) | Check PID file, restart if dead |
| `/loop` check-in | No (dies with session) | `/benchmark_run resume` restarts it |
| `.running` signal | Yes (file) | Tells next session "something is in flight" |
| `.complete` signal | Yes (file) | Tells next session "analysis needed" |
| `.failed` signal | Yes (file) | Tells next session "investigation needed" |

On any new session start where benchmark_data/monitors/*.running exists, the agent should suggest: "A benchmark run is in flight. Use `/benchmark_run resume` to reconnect."

## Safety Rules

```
NEVER launch without an approved plan
NEVER launch without a guardrail script — no exceptions
NEVER launch if another run is active (.running file exists)
NEVER kill execution scripts — only the user or guardrail can abort
NEVER modify the plan or scripts after launch
NEVER make analytical conclusions — that's /benchmark_agent's job
ALWAYS launch guardrail FIRST, then monitor, then execution
ALWAYS verify ALL THREE PIDs are alive before confirming launch
ALWAYS restart guardrail immediately if found dead during resume
ALWAYS preserve all log files even from failed/killed runs
```
